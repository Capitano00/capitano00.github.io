# Boyong Wang Personal Innovation Portfolio

Public GitHub Pages site for Boyong Wang's recruiter/interviewer-facing portfolio, led by the HelioFabric flagship case study.

The site is designed as a company-safe development record: it introduces the personal profile layer first, then uses HelioFabric as evidence of responsible AI-assisted delivery practice, product judgement, governed workflow tooling, and reviewed high-level progress without exposing confidential project data.

## Public Route

Primary route: GitHub Pages.

```text
https://capitano00.github.io
```

Target repository:

```text
Capitano00/capitano00.github.io
```

The GitHub Pages repository should contain approved public site source only. Keep local weekly drafts, review notes, raw screenshots, raw brand source boards, secrets, local data stores, and HelioFabric source code out of the public repo.

## Local Commands

```powershell
npm install
npm run dev
npm run build
```

The static build output is generated in `dist/`.

## Publication Checklist

Before pushing or publishing:

1. Run `npm run build`.
2. Search the tracked public repo for blocked content: client names, sensitive location details, machine-specific paths, secret files, private folders, raw evidence artefacts, raw report/workbook names, and unreviewed update wording.
3. Inspect the generated `dist/` page on desktop and mobile.
4. Confirm GitHub profile and LinkedIn links work.
5. Push only approved public site files to `Capitano00/capitano00.github.io`.

## Brand Assets

Raw brand source boards and design-review notes stay local unless separately approved.

## Disclosure Rules

- Use UK English for public prose.
- Do not publish client names, private project details, sensitive location details, private economics, secrets, raw evidence artefacts, local data stores, source folders, or employer-confidential material.
- Treat screenshots as synthetic, sanitised, or manually reviewed evidence only.
- Keep funding/adoption wording conservative until an approved public statement exists.
- Present HelioFabric as an internal-platform case study, not external SaaS or an open-source code release.
